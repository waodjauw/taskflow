## 三端响应式是如何实现的

### 断点定义

只需一个断点，定义在 `src/composables/useDevice.js`：

| 档位 | 断点 | 标识 |
|------|------|------|
| 手机 | < 768px | `isMobile` |
| 桌面/平板 | ≥ 768px | — |

`useDevice.js` 是**模块级单例**——`matchMedia` 监听器只创建一次，所有组件调用 `useDevice()` 返回同一份响应式引用，外部 `readonly` 只读。

平板和桌面的差异不需要 JS 介入，由 `mobile.css` 里的媒体查询处理——这是纯视觉调整（侧栏宽度、列数），不需要单独的组件树。

### 布局切换（App.vue）

桌面和平板用同一套组件树，手机用另一套，`v-if` 控制：

```
App.vue
  ├── v-if="isMobile"  →  MobileLayout（手机专属组件树）
  └── v-else           →  TopNav + Sidebar + ContentArea（桌面/平板）
```

两套树的差异：

| | 桌面 + 平板 | 手机 |
|--|-----------|------|
| 导航 | `TopNav`（顶部栏） | `MobileTopNav`（汉堡菜单） |
| 侧栏 | `Sidebar`（常驻 260px） | `MobileDrawer`（滑出抽屉） |
| 统计 | `StatsRow`（横向排列） | `MobileStatsRow`（横向滚动） |
| 筛选 | `Toolbar`（内联） | `MobileToolbar` + `MobileFilterSheet`（底部弹出） |
| 任务区 | `TaskGrid`（多列网格） | `MobileTaskGrid`（单列列表） |

**为什么用两套组件树而不是纯 CSS 隐藏？** 手机端交互模式完全不同——没有右键菜单，改用长按；没有侧栏，改用抽屉；筛选从内联改成底部弹出 sheet。用 `v-if` 切换意味着只挂载当前设备需要的组件，另一棵树的组件完全不渲染，节省性能。

以下组件始终挂载在两棵树之外，桌面和手机共享：
- `LockScreen`、`ContextMenu`、`ToastContainer`
- `TaskModal`、`SettingsModal`、`ProgressModal`、`PinChangeModal`
- `AITaskBreakdown`、`AIWeeklyReport`
- `TaskCard`（手机端通过 500ms `touchstart` 长按触发 ContextMenu）

### 平板的 CSS 覆盖

平板复用桌面组件树，只通过 `mobile.css` 里的媒体查询缩小尺寸：

```css
@media (min-width: 768px) and (max-width: 1023px) {
  .sidebar   { width: 200px; }                         /* 桌面 260 → 200 */
  .task-grid { grid-template-columns: repeat(2, 1fr); } /* 3 列 → 2 列 */
  .stats-row { grid-template-columns: repeat(3, 1fr); } /* 5 列 → 3 列 */
  .form-row  { grid-template-columns: 1fr; }            /* 表单列堆叠 */
}
```

### 响应式数据流

```
窗口大小变化
  → matchMedia 触发 change 事件
  → isMobile / isTablet / isDesktop 响应式更新
  → App.vue 的 v-if 自动切换组件树
  → 目标组件树挂载，另一棵树卸载
```
